Responsive Web Design

Slide 2–3

Theory

  • Responsive Web Design allows web pages to adapt to different screen sizes.
  • By default, elements follow normal HTML flow with no floating or positioning.
  • The three main components are:
    • Fluid Layout
    • Viewport Metadata
    • Media Queries
  • Scalable images adjust to the size of their container.

Fluid Layout and Floating

Slide 4–10

Theory

  • By default, elements appear in order with no wrapping.
  • The float property allows elements to wrap around content.
  • The clear property controls floating behavior.
  • Fluid layouts use percentages instead of fixed pixels.
  • Formula: (target ÷ context) × 100

Text Columns

Slide 12

Theory

  • Browsers display text in a single column by default.
  • CSS3 allows multi-column layouts.
  • column-count defines number of columns.
  • column-gap defines spacing.
  • column-span allows elements to span across columns.

Positioning

Slide 13

Theory

  • position property defines element placement.
  • static: default position.
  • relative: relative to original position.
  • absolute: relative to nearest positioned ancestor.
  • fixed: relative to viewport.
  • top, bottom, left, right define placement.

Viewport

Slide 15

Theory

  • The viewport is the visible area of a webpage.
  • It is smaller on mobile devices.
  • Meta viewport controls scaling and layout.
  • width=device-width matches screen width.
  • initial-scale controls zoom level.

Example

<meta name="viewport" content="width=device-width, initial-scale=1">

Media Queries

Slide 16–20

Theory

  • Media queries apply CSS based on device conditions.
  • media types include screen, print, all.
  • Common features include min-width and max-width.
  • They enable responsive layouts.

Examples

@media screen and (max-width:600px){
 body { background: green; }
}
